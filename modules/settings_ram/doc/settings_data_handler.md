# Settings Data Handler 模块使用说明

## 模块概述

`settings_data_handler` 是一个设置数据处理器模块，用于从缓冲RAM读取设置数据，进行验证，并写入到设置存储模块。该模块通过一个启动信号触发，仅执行一次读取-验证-写入流程。

## 数据格式

模块从输入缓冲区读取两个 32 位字（Word）：

| 字偏移 (Word Offset) | 内容 | 说明 |
|---|---|---|
| 0 | 命令字 (Command) | 1=最大行数, 2=最大列数, 3=数据最小值, 4=数据最大值, 5=倒计时时间 |
| 1 | 数据字 (Data) | 32位整数值 |

**UART 输入示例**:
用户输入 `1 5`，表示将最大行数设置为 5。
用户输入 `5 10`，表示将倒计时时间设置为 10 秒。

## 端口说明

### 输入端口

| 端口名 | 位宽 | 说明 |
|--------|------|------|
| `clk` | 1 | 系统时钟 |
| `rst_n` | 1 | 异步复位信号（低电平有效） |
| `start` | 1 | 启动信号（一周期脉冲） |
| `buf_rd_data` | 32 | 缓冲 RAM 读取的数据 (32-bit) |

### 输出端口

| 端口名 | 位宽 | 说明 |
|--------|------|------|
| `busy` | 1 | 忙状态标志 |
| `done` | 1 | 完成信号（一周期脉冲） |
| `error` | 1 | 错误标志（持续直到复位） |
| `buf_rd_addr` | 11 | 缓冲 RAM 读地址 |
| `settings_wr_en` | 1 | 设置写使能 |
| `settings_max_row` | 32 | 最大行数输出 |
| `settings_max_col` | 32 | 最大列数输出 |
| `settings_data_min` | 32 | 数据最小值输出 |
| `settings_data_max` | 32 | 数据最大值输出 |
| `settings_countdown` | 32 | 倒计时时间输出 |

## 验证规则

模块会对读取的数据进行以下验证：

### 1. 命令有效性
- 命令字必须为 1, 2, 3, 4, 或 5
- 其他值视为无效命令

### 2. 行数/列数限制
- 最大行数（命令=1）：必须在 1-32 范围内
- 最大列数（命令=2）：必须在 1-32 范围内
- 值为 0 或超过 32 均视为无效

### 3. 数据范围限制
- 数据最小值（命令=3）：无特殊限制
- 数据最大值（命令=4）：不可超过 65535

### 4. 倒计时时间限制
- 倒计时时间（命令=5）：必须在 5-15 范围内

## 状态机流程

```
IDLE → READ_CMD → WAIT_CMD → READ_DATA → WAIT_DATA → VALIDATE
                                                        ↓
                                                [验证成功]
                                                        ↓
IDLE ← WRITE_SETTINGS ←─────────────────────────────────┘
  ↑
  └─── [验证失败，设置error]
```

## 错误处理

### 错误标志行为
- `error` 信号在检测到错误时拉高
- **持续保持高电平直到系统复位**
- 一旦 `error` 为高，模块拒绝新的 `start` 请求
- 必须通过复位清除错误状态

### 错误场景
1. **无效命令**：命令字不在 1-5 范围内
2. **行数越界**：行数为0或超过32
3. **列数越界**：列数为0或超过32
4. **数据值越界**：数据最大值超过65535
5. **倒计时越界**：倒计时时间不在 5-15 范围内
