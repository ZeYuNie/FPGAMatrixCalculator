# 数据块预处理单元 TTU (`tile_transform_unit.sv`)

规格如下：

1. 适用 $F(4\times4,3\times3)$ 的 Winograd 卷积算法
2. 输入输出均固定为 $6\times6$
3. 为了速度考虑，进行的是整数乘法，因此输出结果被扩大了 $4\times4=16$ 倍，后续需要缩放回来
4. 所有的运算都是 16 位宽，注意值域不要超出数据范围
5. 流水线设计，共有 3 个状态，即 3 个时钟周期完成一次数据块预处理，transform_done 为高时表示当前数据块预处理完成，可以读取输出数据

## 行为级仿真

`tile_transform_unit_sim.sv`，测试五种数据块输入，测试均通过

## 时序级仿真

经过测试，在 200 MHz 下可以以约 1.3ns 的余量工作
